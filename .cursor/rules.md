# Cursor 프로젝트 규칙

이 프로젝트는 Kotlin + Jetpack Compose 로 만드는 안드로이드 모바일 게임이다.  
Cursor는 아래 규칙을 반드시 따르도록 한다.

---

# 🌏 언어 설정

**사용자는 한국어를 선호한다.**

- 모든 문서는 한국어로 작성한다.
- 코드 주석도 한국어로 작성한다.
- 사용자와의 대화는 한국어로 한다.
- 단, 코드의 변수명, 함수명, 클래스명은 영어로 작성한다.

---

# 🧭 기본 원칙

- 항상 MVP 범위를 유지한다.
- 요청되지 않은 기능을 임의로 추가하지 않는다.
- 복잡한 설계보다 단순하고 읽기 쉬운 코드를 우선한다.
- 과도한 추상화 및 과설계를 피한다.

이 프로젝트의 목표는 **출시 가능한 게임을 만드는 것**이다.

---

# 🛠 기술 스택

반드시 아래 기술을 사용한다.

- Kotlin
- Jetpack Compose
- Android 앱 프로젝트

명시적으로 요청되지 않는 한 절대 추가하지 말 것:

- 백엔드
- 네트워크 통신
- 데이터베이스
- 멀티플레이 기능

이 게임은 **오프라인 게임**이다.

---

# 🏗 아키텍처 규칙

MVP 단계에서는 단순한 구조를 유지한다.

레이어 구조:
- ui
- domain
- data (필요할 때만)

금지 사항:
- 멀티 모듈 구조 도입 금지
- 복잡한 클린 아키텍처 강제 적용 금지

현재는 **단일 app 모듈만 사용**한다.

---

# 🎨 UI 규칙 (Compose)

- UI는 반드시 Jetpack Compose 로 작성한다.
- XML 레이아웃 사용 금지.
- 가능한 Stateless Composable 을 사용한다.
- Composable 은 작고 재사용 가능하게 작성한다.
- 성급한 최적화 금지.

---

# 🎮 게임 개발 규칙

이 프로젝트는 일반 앱이 아니라 **모바일 게임**이다.

우선순위:

1️⃣ 플레이 가능하게 만들기  
2️⃣ 그 다음 폴리싱  
3️⃣ 그 다음 기능 추가

엔터프라이즈 앱 패턴 적용 금지.

---

# 🧾 코드 스타일

- Gradle 스크립트는 Kotlin DSL 사용.
- Android / Kotlin 공식 스타일 가이드 준수.
- 불필요한 라이브러리 추가 금지.

---

# 🤖 Cursor 작업 방식 규칙

Cursor는 작업 시작 시 항상:

1. 먼저 작업 계획을 간단히 설명한다.
2. 그 다음 코드를 작성한다.

설명 없이 큰 코드 블록을 한 번에 생성하지 않는다.

## 📄 문서 작성 규칙

**문서는 사용자가 명시적으로 요청할 때만 작성한다.**

- 사용자가 "문서 작성해줘", "README 만들어줘" 등 명시적으로 요청하지 않으면 문서를 생성하지 않는다.
- 코드 수정이나 버그 수정 시 자동으로 문서를 만들지 않는다.
- 작업 완료 후 요약 설명은 괜찮지만, 별도 마크다운 파일을 생성하지 않는다.
